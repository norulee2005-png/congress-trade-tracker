# External Integrations

**Analysis Date:** 2026-04-01

## APIs & External Services

**Congressional Disclosures - House:**
- House Financial Disclosures (disclosures-clerk.house.gov)
  - SDK/Client: axios + xml2js
  - Purpose: Fetch House Periodic Transaction Reports (PTRs) for stock trading disclosures
  - Implementation: `src/lib/house-scraper.ts`, `src/lib/house-pdf-scraper.ts`
  - Data: House members' names, filing IDs, transaction details (ticker, date, amount, type)
  - Rate limit: Polite 200ms delay between requests

**Congressional Disclosures - Senate:**
- Senate eFDS (Elasticsearch Financial Disclosure System) - efts.senate.gov
  - SDK/Client: axios + xml2js
  - Purpose: Fetch Senate Periodic Transaction Reports via search API
  - Implementation: `src/lib/senate-scraper.ts`
  - Data: Senator names, tickers, transaction dates, amounts, transaction types
  - Rate limit: 500ms delay between paginated requests
  - API: JSON Elasticsearch endpoint at `https://efts.senate.gov/LATEST/search-index`

**Stock Price Data:**
- Financial Modeling Prep (FMP) API - Primary
  - SDK/Client: axios
  - Auth: `FMP_API_KEY` (free tier: 250 req/day)
  - Implementation: `src/lib/stock-price-fetcher.ts`
  - Endpoint: `/api/v3/quote/{tickers}` (comma-separated, max 50 per batch)
  - Caching: Module-level cache with 5-min TTL, max 500 entries
  
- Yahoo Finance v8 - Fallback
  - SDK/Client: axios + fetch
  - Implementation: `src/lib/stock-price-fetcher.ts` (fallback for missing tickers)
  - Endpoint: `/v8/finance/chart/` with interval=1d&range=1d
  - No auth required

## Data Storage

**Databases:**
- Neon PostgreSQL
  - Connection: `DATABASE_URL` env var (serverless connection string)
  - Client: @neondatabase/serverless + drizzle-orm
  - Implementation: `src/db/db-client.ts`
  - Schema: `src/db/schema/` (users, politicians, trades, stocks, alerts, alert-deliveries, magic-links)
  - Migrations: `drizzle/migrations/` (auto-generated by drizzle-kit)

**File Storage:**
- None detected (static blog content via `src/content/` markdown files, OG images generated dynamically)

**Caching:**
- In-memory stock price cache (module-level in `src/lib/stock-price-fetcher.ts`)
- Server-side session tokens via JWT (no Redis/external cache layer)

## Authentication & Identity

**Auth Provider:**
- Custom JWT-based magic link authentication
  - Implementation: `src/lib/auth-session.ts`, `src/lib/magic-link-service.ts`
  - Token creation: HS256 signed JWT with 7-day expiry
  - Session storage: HTTP-only cookie (`session`)
  - Magic link delivery: Email via Resend
  - Routes: 
    - `src/app/api/auth/send-magic-link/route.ts` - Initiate magic link
    - `src/app/api/auth/verify/route.ts` - Verify JWT and set session
    - `src/app/api/auth/signout/route.ts` - Clear session cookie

- Auth: `RESEND_API_KEY`
